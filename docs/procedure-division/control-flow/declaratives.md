# DECLARATIVES and USE

The `DECLARATIVES` section defines procedures that are invoked automatically by the runtime in response to specific events, rather than by explicit CALL or PERFORM. Each declarative section begins with a `USE` statement that specifies the triggering condition.

- **Standard:** COBOL-68, COBOL-85, COBOL 2002, COBOL 2014, COBOL 2023
- **Division:** Procedure Division
- **Category:** Control Flow / Error Handling

!!! info "Historical Note"
    DECLARATIVES with USE statements have been part of COBOL since the earliest
    specifications. The AspectCobol study (Lammel & De Schutter, 2005) identified
    them as arguably the oldest aspect-oriented programming construct — they
    provide automatic interception of I/O operations without the intercepted
    code being aware of the handler.

---

## Syntax

```cobol
PROCEDURE DIVISION.
DECLARATIVES.

section-name-1 SECTION.
    USE AFTER STANDARD { ERROR | EXCEPTION } PROCEDURE ON
        { file-name-1 | INPUT | OUTPUT | I-O | EXTEND }.
paragraph-name-1.
    [imperative statements]

[ section-name-2 SECTION.
    USE ... ]

END DECLARATIVES.

main-section SECTION.
    [main program logic]
```

The DECLARATIVES section must appear immediately after the PROCEDURE DIVISION header and before any non-declarative code.

---

## USE AFTER STANDARD ERROR PROCEDURE

Specifies error-handling procedures that are automatically invoked when a file I/O error occurs.

```cobol
USE AFTER STANDARD { ERROR | EXCEPTION } PROCEDURE ON
    { file-name | INPUT | OUTPUT | I-O | EXTEND }
```

### Rules

1. The declarative section is invoked automatically whenever an I/O statement (OPEN, READ, WRITE, CLOSE, REWRITE, DELETE, START) fails on the specified file or category of files.
2. `ON file-name` applies to a specific file.
3. `ON INPUT` applies to all input files not named in another USE.
4. `ON OUTPUT` applies to all output files not named in another USE.
5. `ON I-O` applies to all files opened I-O not named in another USE.
6. `ON EXTEND` applies to all files opened EXTEND not named in another USE.
7. A file-specific USE takes precedence over a category-level USE.
8. The `FILE STATUS` variable should be examined within the declarative to determine the specific error.
9. After the declarative executes, control returns to the statement following the one that caused the error.

### Example

```cobol
       PROCEDURE DIVISION.
       DECLARATIVES.

       INPUT-ERROR SECTION.
           USE AFTER STANDARD ERROR PROCEDURE ON INPUT.
       INPUT-ERROR-PARA.
           DISPLAY "Input error on file, status: "
               WS-FILE-STATUS
           IF WS-FILE-STATUS = "35"
               DISPLAY "File not found"
           END-IF.

       MASTER-ERROR SECTION.
           USE AFTER STANDARD ERROR PROCEDURE ON
               MASTER-FILE.
       MASTER-ERROR-PARA.
           EVALUATE WS-MASTER-STATUS
               WHEN "23"
                   DISPLAY "Record not found"
               WHEN "22"
                   DISPLAY "Duplicate key"
               WHEN OTHER
                   DISPLAY "Master file error: "
                       WS-MASTER-STATUS
                   STOP RUN
           END-EVALUATE.

       END DECLARATIVES.

       MAIN-PROCESSING SECTION.
       MAIN-PARA.
           OPEN INPUT TRANS-FILE
                I-O   MASTER-FILE
           PERFORM PROCESS-RECORDS
               UNTIL WS-EOF
           CLOSE TRANS-FILE MASTER-FILE
           STOP RUN.

       PROCESS-RECORDS.
           READ TRANS-FILE
               AT END SET WS-EOF TO TRUE
           END-READ
      *>   If READ fails, INPUT-ERROR is invoked automatically
      *>   If a MASTER-FILE operation fails, MASTER-ERROR
      *>   is invoked automatically
           ...
```

## USE BEFORE REPORTING

Specifies procedures invoked before a report group is generated by the Report Writer.

- **Standard:** COBOL-85 (Report Writer module, optional)

```cobol
USE BEFORE REPORTING report-group-name
```

This allows customization of report data immediately before output — for example, calculating subtotals or formatting fields conditionally.

```cobol
       REPORT-HEADING-PROC SECTION.
           USE BEFORE REPORTING PAGE-HEADER.
       REPORT-HEADING-PARA.
           MOVE FUNCTION CURRENT-DATE(1:8)
               TO RPT-DATE.
```

## USE FOR DEBUGGING (Removed)

Specified procedures invoked when a data item was modified or a procedure was executed, providing runtime tracing and debugging.

- **Standard:** COBOL-74, obsolete in COBOL-85, removed in COBOL 2002

!!! warning "Removed"
    `USE FOR DEBUGGING` was marked obsolete in COBOL-85 and removed in
    COBOL 2002. It required `WITH DEBUGGING MODE` in the SOURCE-COMPUTER
    paragraph. Modern debuggers and compiler options have replaced this
    facility entirely.

```cobol
USE FOR DEBUGGING ON { data-name | procedure-name | ALL PROCEDURES }
```

When triggered, the special register `DEBUG-ITEM` contained information about the event:

| Field | Content |
|-------|---------|
| DEBUG-LINE | Source line number |
| DEBUG-NAME | Name of the item or procedure |
| DEBUG-CONTENTS | Current value (for data items) |

## USE AFTER EXCEPTION CONDITION (COBOL 2002)

Handles structured exception conditions identified by EC-xxx exception names.

```cobol
USE AFTER EXCEPTION CONDITION exception-name
```

This integrates with the COBOL 2002 exception handling model (RAISE, RESUME, exception conditions).

---

## General Rules

1. Declarative sections **cannot** be PERFORMed or branched to (via GO TO) from non-declarative code.
2. Non-declarative procedures **cannot** be PERFORMed from within a declarative section. Use CALL for external invocation.
3. Each USE statement must be the first and only statement in its section header.
4. Multiple USE sections can exist within a single DECLARATIVES block.
5. A declarative procedure may contain any statements except those that would reference another declarative section.
