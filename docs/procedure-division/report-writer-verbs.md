# Report Writer Verbs (INITIATE, GENERATE, TERMINATE)

The Report Writer module provides declarative report generation through three Procedure Division verbs and the USE BEFORE REPORTING declarative, working with report descriptions (RD entries) in the Data Division.

- **Standard:** COBOL-68, COBOL-85 (optional module), COBOL 2002, COBOL 2014
- **Division:** Procedure Division
- **Category:** Report Writer

!!! note "Optional Module"
    Report Writer is an optional module in the COBOL standard. Not all
    compilers support it. IBM Enterprise COBOL, Micro Focus, and GnuCOBOL
    provide Report Writer support.

---

## INITIATE

Begins report processing for one or more reports.

```cobol
INITIATE report-name-1 [ report-name-2 ] ...
```

### Rules

1. The file associated with the report must be opened before INITIATE.
2. INITIATE sets PAGE-COUNTER to 1 and LINE-COUNTER to 0.
3. All SUM counters are set to zero.
4. No page or report headings are generated by INITIATE — they are generated on the first GENERATE.

```cobol
OPEN OUTPUT REPORT-FILE
INITIATE SALES-REPORT
```

---

## GENERATE

Produces report output — either a detail line or a summary report event.

```cobol
GENERATE { report-name | data-name }
```

### Rules

1. `GENERATE data-name` produces a detail report group (the named detail line) and any associated control headings, control footings, and page headings/footings as needed.
2. `GENERATE report-name` produces a **summary report** — it processes control breaks and SUM counters without producing detail lines.
3. On the first GENERATE, Report Writer automatically produces the REPORT HEADING and PAGE HEADING groups.
4. When a control break is detected (data item in the CONTROL clause changes value), Report Writer automatically produces CONTROL FOOTING groups for the completed level and CONTROL HEADING groups for the new level.
5. When the page body is full (LINAGE-COUNTER reaches the FOOTING value), Report Writer produces PAGE FOOTING and PAGE HEADING groups and advances to a new page.

```cobol
PERFORM UNTIL WS-EOF
    READ INPUT-FILE
        AT END SET WS-EOF TO TRUE
    END-READ
    IF NOT WS-EOF
        MOVE INPUT-DATA TO REPORT-DETAIL-FIELDS
        GENERATE DETAIL-LINE
    END-IF
END-PERFORM
```

---

## TERMINATE

Ends report processing, producing final control footings and the report footing.

```cobol
TERMINATE report-name-1 [ report-name-2 ] ...
```

### Rules

1. TERMINATE produces all remaining CONTROL FOOTING groups (for the final control break), the REPORT FOOTING group, and the final PAGE FOOTING.
2. After TERMINATE, the file must be closed.
3. TERMINATE does not close the file — an explicit CLOSE is required.

```cobol
TERMINATE SALES-REPORT
CLOSE REPORT-FILE
```

---

## SUPPRESS

Suppresses printing of the current report group during a USE BEFORE REPORTING procedure.

```cobol
SUPPRESS PRINTING
```

Can only appear within a USE BEFORE REPORTING declarative section.

## Complete Example

```cobol
       DATA DIVISION.
       FILE SECTION.
       FD  REPORT-FILE.
       01  REPORT-LINE     PIC X(80).

       REPORT SECTION.
       RD  SALES-REPORT
           CONTROLS ARE FINAL DEPT-CODE
           PAGE LIMIT IS 60 LINES
           HEADING 1
           FIRST DETAIL 5
           LAST DETAIL 55
           FOOTING 58.

       01  TYPE IS REPORT HEADING.
           05  LINE 1.
               10  COLUMN 30  PIC X(20) VALUE "SALES REPORT".

       01  TYPE IS PAGE HEADING.
           05  LINE 3.
               10  COLUMN 1   PIC X(4)  VALUE "DEPT".
               10  COLUMN 10  PIC X(8)  VALUE "EMPLOYEE".
               10  COLUMN 25  PIC X(6)  VALUE "AMOUNT".
           05  LINE 4.
               10  COLUMN 1   PIC X(40) VALUE ALL "-".

       01  DETAIL-LINE TYPE IS DETAIL.
           05  LINE PLUS 1.
               10  COLUMN 1   PIC X(5)  SOURCE DEPT-CODE.
               10  COLUMN 10  PIC X(20) SOURCE EMP-NAME.
               10  COLUMN 25  PIC $ZZ,ZZ9.99
                               SOURCE SALE-AMOUNT.

       01  TYPE IS CONTROL FOOTING DEPT-CODE.
           05  LINE PLUS 2.
               10  COLUMN 10  PIC X(12) VALUE "Dept Total:".
               10  COLUMN 25  PIC $ZZZ,ZZ9.99
                               SUM SALE-AMOUNT.

       01  TYPE IS CONTROL FOOTING FINAL.
           05  LINE PLUS 2.
               10  COLUMN 10  PIC X(13) VALUE "Grand Total:".
               10  COLUMN 25  PIC $Z,ZZZ,ZZ9.99
                               SUM SALE-AMOUNT.

       PROCEDURE DIVISION.
           OPEN INPUT SALES-FILE OUTPUT REPORT-FILE
           INITIATE SALES-REPORT
           PERFORM UNTIL WS-EOF
               READ SALES-FILE
                   AT END SET WS-EOF TO TRUE
               END-READ
               IF NOT WS-EOF
                   GENERATE DETAIL-LINE
               END-IF
           END-PERFORM
           TERMINATE SALES-REPORT
           CLOSE SALES-FILE REPORT-FILE
           STOP RUN.
```
